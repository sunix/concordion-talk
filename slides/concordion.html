## TDD and BDD
### TDD Test Driven Development
Write test first, implements after
###Benefits - Developer natural APIs - Easier to refactor - Nice % test coverage - No over code design
### BDD Behaviour Driven Development
Write Human readable test scenario first
###Benefits - Human readable (dev, PO, users ...) - Nice % Acceptance tests, Non regression tests coverage - Developer knows exactly what to implement ####Behavioral specification - User stories - Scenarios - Given - When - Then

###Concordion the basics - Behavioral Specs/scenario in html - Fixtures in java #### Concordion life cycle - Linked all together with special attributes in the specs. - Behavioral specs is parsed - Execute fixtures instructions - Generates reports

Concordion basics

####Behavioral specs and fixtures - same folder/package - same name
####Requirements - Concordion namespace - `ConcordionRunner` JUnit4 runner
####Basic Concordion instrumentations - html tags drive fixture methods
####Use anything in java to "drive" your test - Plain java - Selenium Webdriver - SWTBot - Apache commons exec
